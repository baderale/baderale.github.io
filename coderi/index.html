Coderi.

async fn code(note) -> "ICD-10 + rationale + source sentence"

A clinical-coding co-pilot that sits between a clinic's EHR and its billing software. It de-identifies notes, extracts clinical entities, retrieves candidate ICD-10 codes, and proposes a grounded code for a human coder to approve — running on the clinic's own isolated infrastructure, never a third-party AI API.

moat.md

why it's different

vs. Fathom / CodaMetrix / Nym — the load-bearing premises, not preferences

Self-owned model, never a third-party AI

Patient data stays inside the clinic's isolated cloud tenant. No clinical text ever reaches an external LLM API.

Context-aware

Distinguishes active vs. historical, laterality & acuity — the incumbents' #1 error mode, handled by a dedicated verify pass.

Explainable

Every code carries a rationale + the source sentence — audit and appeal gold, not a black box.

Human-in-the-loop co-pilot

The pipeline proposes; a human approves or overrides. Lower liability, easier trust, faster sales.

Grounded, never hallucinated

Codes come from retrieval over a local code DB. The model selects and justifies — it cannot fabricate a code.

Outpatient-focused

Priced and scoped for the ambulatory practices the over-built incumbents leave underserved.

architecture.json

three planes

PHI is walled into a per-tenant boundary; the shared cloud only ever sees de-identified text

PlaneResponsibilitySees PHI?Where it runs
Data plane de-identify → extract → retrieve → map → verify → report Yes per-tenant isolated AWS VPC · Coderi's own model
Control plane dashboard, auth, billing, tenant provisioning No (de-id only) shared multi-tenant cloud SaaS
Connectors EHR (FHIR R4) in, coded claim out Yes (transit) per-tenant

Orchestrated in LangGraph with a real interrupt() human gate, durable across processes. Single-tenant cloud VPC on AWS (account-per-tenant via Organizations + Control Tower); each clinic runs Coderi's own model under a BAA. BYOC into the customer's own AWS account is an enterprise upsell.

flywheel.md

the training flywheel

Teacher maps today; every human approval trains the owned student coder tomorrow

1

Teacher

A capable local LLM (MedGemma 1.5 4B via Ollama) does retrieval-grounded ICD-10 mapping today — no training data required to start.

2

Capture

Every human-approved or corrected code becomes a labeled note↔code pair — the proprietary dataset that compounds.

3

Student

QLoRA fine-tune a small specialist coder on MIMIC-IV distant supervision + captured pairs → fast, cheap, CPU-servable, fully owned. The teacher then handles only ambiguous cases.
